# Support Guide for Axon

Thank you for using Axon\! We want to ensure you have the best experience building signal-native state machines.

Please follow these guidelines to get help.

## 🛠 Official Support Channel: GitHub Issues

We use **GitHub Issues** as our primary and only support channel. To keep the project maintainable, we do not provide support via email, Twitter (X), or private messages.

### [Open a Support Request](https://www.google.com/search?q=https://github.com/your-username/axon/issues/new/choose)

-----

## 🔍 Before You Open an Issue

1.  **Check the Documentation**: Most common patterns and API details are covered in our [README.md](https://www.google.com/search?q=./README.md) and `/docs` folder.
2.  **Search Existing Issues**: Your question may have already been answered. Search both **Open** and **Closed** issues.
3.  **Update Axon**: Ensure you are running the latest version of the library.

-----

## 📝 How to Ask for Support

To help us help you, please provide the following in your issue:

  * **A Clear Description**: What are you trying to achieve, and what is the current blocker?
  * **Code Snippet**: Show us your `AxonGraph` and how you are initializing the `Axon` instance.
  * **Environment Info**:
      * Axon Version
      * Angular Version
      * TypeScript Version
  * **Minimal Reproduction**: If possible, provide a link to a [StackBlitz](https://www.google.com/search?q=https://stackblitz.com) or a minimal GitHub repo. Requests with reproductions are almost always resolved faster.

-----

## ⏳ What to Expect

Axon is an open-source project maintained by the community.

  * **Response Time**: We aim to acknowledge support requests within **3–5 business days**, but this may vary based on maintainer availability.
  * **Resolution**: We will do our best to point you in the right direction. If your support request turns out to be a bug, we will convert it into a Bug Report.

## 🚫 What is NOT Support?

  * **General Angular Questions**: If your question is about Angular Signals in general rather than Axon specifically, please use [Stack Overflow](https://www.google.com/search?q=https://stackoverflow.com/questions/tagged/angular) or the Angular Discord.
  * **Custom Implementation Services**: We cannot write your entire application logic for you, but we can help you understand how Axon fits into it.

-----

### Pro-Tip: Labeling

When you open a support request, if you have the permissions, please tag it with the `question` or `help wanted` label. This helps maintainers prioritize bugs (which break things) vs. questions (which clarify things).
